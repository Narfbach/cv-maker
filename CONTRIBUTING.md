# Contributing to CV Maker

Thank you for considering contributing to CV Maker. This document outlines the process and guidelines for contributing.

## Development Setup

1. Fork and clone the repository
2. Open `index.html` in your browser or use a local server
3. Make your changes
4. Test thoroughly across different browsers

## Code Style Guidelines

### JavaScript
- Use ES6+ syntax
- Follow functional programming principles where applicable
- Keep functions small and focused
- Use meaningful variable names
- Add comments for complex logic

### CSS
- Use CSS custom properties for theming
- Follow BEM-like naming conventions
- Maintain mobile-first responsive design
- Keep specificity low

### HTML
- Use semantic HTML5 elements
- Ensure accessibility (ARIA labels where needed)
- Maintain clean indentation

## Testing Checklist

Before submitting a pull request, ensure:

- [ ] Code works in Chrome, Firefox, and Safari
- [ ] Responsive design works on mobile devices
- [ ] Print/PDF export functions correctly
- [ ] No console errors or warnings
- [ ] All features work without internet connection
- [ ] Code follows existing style patterns

## Commit Messages

Use clear, descriptive commit messages:
- `feat: add new customization option`
- `fix: resolve PDF export issue`
- `docs: update README installation steps`
- `style: improve button hover effects`
- `refactor: optimize render performance`

## Pull Request Process

1. Update README.md if adding new features
2. Ensure all tests pass
3. Update version number if applicable
4. Describe changes clearly in PR description
5. Link any related issues

## Feature Requests

Open an issue with:
- Clear description of the feature
- Use case and benefits
- Potential implementation approach

## Bug Reports

Include:
- Browser and version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

## Questions

For questions, open a GitHub issue with the "question" label.
